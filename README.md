Davids System Monitor

=====================


Description:

------------

I made Davids System Monitor. It is a program that runs in the terminal. It shows you what your computer is doing. You can see how much of your computers brain, memory and disk space is being used. The program is easy to read. It uses colors to show you what is going on. It works on Windows computers. It is safe to use. You do not need a graphics card to use it.

Features:

---------

- The program updates in time

- It shows you how much of your computers brain is being used

- It shows you how much of your computers memory is being used

- It shows you how much of your disk space is being used

- It uses colors to make it easy to see what is going on

. Green means things are okay

. Yellow means things are getting busy

. Red means things are very busy

- The program updates smoothly

- You can stop the program by pressing Ctrl+C

Requirements:

-------------

- You need to have Python 3.10 or later on your computer

- You need to have the psutil library

You can get it by typing this in the terminal:

pip install psutil

Optional Enhancements:

---------------------

- You can add a feature to monitor your graphics card but you need to have a special graphics card

- You can add a feature to monitor your computers temperature but you need to have some extra programs, on your computer

- You can add a feature to save the data to a file so you can look at it later

- You can add a feature to show you what all of your computers brains are doing

Usage:

------

1. Open the terminal on your computer.

2. Go to the folder where Davids System Monitor's

cd path\to\davids_system_monitor

3. Run the program:

python system_monitor.py

4. Press Ctrl+C to stop the program.

Notes:

------

- Davids System Monitor works on Windows computers.

- If you use the PowerShell terminal it will look nicer because it can show colors.

- I did not turn on the feature to monitor the graphics card because I want Davids System Monitor to work on all computers.
